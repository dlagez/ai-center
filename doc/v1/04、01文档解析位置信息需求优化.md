# 文档解析位置信息需求优化

## 1. 文档目的

本文档用于补充说明文档解析模块中的“位置信息”需求。

主 PRD 已明确文档解析模块需要为后续 RAG 检索提供原文回溯能力，但考虑到当前阶段应优先控制复杂度，V1 仅要求保留最小可用位置信息。

本文档的目标是：

- 明确当前阶段需要实现的最小位置信息范围
- 说明为什么当前先只做页码与 Excel 行号
- 记录后续可能扩展的位置信息方向

---

## 2. 背景

文档解析结果会继续进入：

- Chunking
- Embedding
- 索引构建
- RAG 检索
- 检索结果引用与原文跳转

如果完全不保留位置信息，后续会出现以下问题：

- 检索命中后无法知道答案来自原文哪一页
- 表格类文档无法知道内容来自哪一行
- 引用展示时只能返回文本，无法返回来源定位

但如果在 V1 就一次性引入过细粒度的位置模型，也会明显提高实现复杂度，尤其是：

- PDF 块级结构抽取
- OCR 坐标统一
- Word 段落锚点
- Excel 单元格精确坐标
- 字符级 offset 回溯

因此，当前阶段建议采用“最小可用定位模型”。

---

## 3. 范围结论

当前阶段，文档解析位置信息先只保留：

- 页式文档的页码 `page_no`
- Excel / CSV 等表格类文档的行号 `row_index`

这是当前版本的约束，不是最终上限。

换句话说：

- 先满足“检索后知道在第几页、哪一行”
- 暂不要求“精确到哪个段落、哪个字符、哪个单元格区域”

---

## 4. 当前阶段需求

### 4.1 页式文档

适用文件类型：

- PDF
- 图片 OCR 结果
- PPT / PPTX
- Word / DOCX

当前阶段要求：

- 解析结果中保留 `page_no`
- Chunk 或检索片段需要能回溯到来源页码

说明：

- 对于 Word、PPT 这类原始格式未必天然具备“页面”概念的文档，若解析器暂时无法稳定给出页码，可先不强制，但对外仍应预留统一字段

### 4.2 表格类文档

适用文件类型：

- XLSX
- CSV

当前阶段要求：

- 解析结果中保留 `row_index`
- 检索片段需要能回溯到来源行号

说明：

- 当前阶段优先保留“第几行”
- 暂不强制要求 `sheet_name`、`col_index`、`cell_ref`

---

## 5. 建议数据结构

当前阶段建议使用统一最小位置对象：

```python
class DocumentLocation(BaseModel):
    page_no: int | None = None
    row_index: int | None = None
```

建议：

- 页式文档只填 `page_no`
- 表格类文档只填 `row_index`
- 非适用字段保持为空

在文档解析结果中建议增加：

```python
class DocumentParseResult(BaseModel):
    ...
    locations: list[DocumentLocation] = Field(default_factory=list)
```

---

## 6. 为什么当前先不做更细粒度

当前先不强制引入以下信息：

- bbox / polygon
- block_id
- paragraph_index
- char_start / char_end
- sheet_name
- col_index
- cell_ref

原因是：

- 这些字段跨文件类型统一难度高
- 不同 parser 的输出稳定性差异较大
- 当前阶段 RAG 最核心的诉求是“先能回源”，不是“先做精确高亮”
- 先把最小定位链路跑通，更符合 V1 节奏

---

## 7. 后续可扩展方向

后续如果业务需要更精细的引用、高亮、原文跳转，可逐步扩展：

- PDF / OCR：`bbox`
- Word：`paragraph_index`
- Excel：`sheet_name`、`col_index`、`cell_ref`
- 通用文本块：`block_id`
- 字符级回溯：`char_start`、`char_end`

这些能力建议在后续版本中逐步引入，而不是在当前版本一次性落地。

---

## 8. 实施建议

当前阶段实施建议如下：

1. 主 PRD 中明确：V1 位置信息最小范围为 `page_no` 与 `row_index`
2. 文档解析结果结构预留 `locations`
3. Chunking 与检索结果元数据同步保留来源页码或来源行号
4. 更细粒度位置模型后续再单独设计和实现

---

## 9. 验收标准

- PDF / 图片 OCR 结果可以返回 `page_no`
- XLSX / CSV 结果可以返回 `row_index`
- 检索结果可以携带来源页码或来源行号
- 当前阶段不要求实现精确坐标高亮
